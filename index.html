README.md

Vegas-born, Utah-trained developer with a background in psychology and behavior analysis.

I started in psychology. After undergrad I spent time doing behavior analysis — which is where I learned to care a lot about whether a system that claims to be working is actually working.

The move to software wasn't really a pivot. The question I was chasing in psych (can the system making decisions actually be trusted to make them?) turned out to be the same question that matters most in software. So now I build and evaluate systems with that question front and center. If I can't measure whether it's working, I don't trust that it is.

My MSD capstone was ML-based anomaly detection on enterprise activity logs. The hard part wasn't the model — it was making the system explain itself instead of handing an analyst a mysterious score. That's what pushed me toward RAG.

I'm currently researching retrieval-augmented generation for scientific computing at the University of Utah — if the system is going to answer for something, it should be able to back it up.
